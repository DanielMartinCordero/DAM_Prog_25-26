# 🧩 Unit 08: Interfaces y Polimorfismo (Interfaces & Polymorphism)

### 🇪🇸 Resumen de la unidad
En esta unidad se profundiza en el concepto de **interfaces**, el nivel más alto de abstracción en Java. El enfoque principal es aprender a definir **contratos de comportamiento** que las clases deben cumplir, permitiendo un diseño de software desacoplado y altamente flexible.

**Conceptos clave aprendidos:**
* **Contratos**: Uso de `interface` e `implements` para obligar a las clases a proporcionar implementaciones específicas.
* **Herencia Múltiple**: Simulación de herencia múltiple mediante la implementación de varias interfaces en una sola clase.
* **Métodos Default y Static**: Uso de métodos con cuerpo dentro de interfaces para proporcionar lógica compartida sin romper la compatibilidad.
* **Interfaces Estándar**: Implementación de interfaces nativas de Java como `Comparable` y `Comparator` para la ordenación de objetos.
* **Polimorfismo de Interfaz**: Tratamiento de objetos diversos bajo un tipo de interfaz común para maximizar la reutilización.
* **Clases Anónimas**: Introducción a la implementación de interfaces al vuelo sin necesidad de crear archivos de clase adicionales.

### 🇺🇸 Unit Overview
This unit explores **Interfaces**, Java's mechanism for achieving the highest level of abstraction. The core objective is to master **behavioral contracts**, ensuring that classes adhere to specific protocols while promoting decoupled and maintainable code.

**Key Technical Skills:**
* **API Design**: Defining clear protocols using the `interface` keyword.
* **Multiple Implementation**: Overcoming single inheritance limitations by implementing multiple interfaces.
* **Modern Interface Features**: Leveraging `default` and `static` methods for base functionality.
* **Standard Protocols**: Mastering built-in interfaces like `Comparable` and `Comparator` for advanced sorting logic.
* **Decoupling**: Utilizing interface types to write more generic and robust code.
* **Functional Basics**: Initial contact with functional interfaces as a bridge to Lambdas and modern Java patterns.
